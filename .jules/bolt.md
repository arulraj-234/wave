## 2024-05-18 - Avoid React.memo on components consuming rapidly-changing contexts
**Learning:** Using `React.memo` on components like `SongCard` is ineffective and an anti-pattern in this codebase because these components use `useContext(PlayerContext)` internally. Since `PlayerContext` updates frequently (e.g. `currentSong`, `isPlaying` state), the components will re-render anyway, completely bypassing the `React.memo` prop check. Furthermore, `ContentCard` frequently receives inline functions as props from its parent (e.g. `onAddToQueue={() => ...}` in `Dashboard`), which causes shallow comparison to fail on every render, adding overhead without benefit.
**Action:** Avoid micro-optimizations like `React.memo` without verifying that props are stable and contexts are isolated. Opt for proven native optimizations like `loading="lazy"` on images instead.
